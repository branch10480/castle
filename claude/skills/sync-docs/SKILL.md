# sync-docs

## 手順

### 1. 現在のプロジェクト構成を取得

Glob や find でプロジェクトのソースファイル一覧を取得する。`.git/`, `DerivedData/`, `node_modules/`, `.build/`, `.claude/` は除外する。

### 2. 既存ドキュメントを読み込む

- `CLAUDE.md` を読む
- `README.md` を読む
- データモデルの実装ファイルがあれば読む（プロパティ名・型の確認用）

### 3. 差分を特定して修正

**CLAUDE.md:**
- ディレクトリ構成ツリーが実際のファイル/フォルダと一致しているか
- 開発フェーズのチェックボックスが進捗と一致しているか
- 新規ファイル/ディレクトリの追加漏れ、削除済みファイルの残存がないか
- 技術方針・規約が実装と矛盾していないか

**README.md:**
- データモデルのコード例が実装と一致しているか（プロパティ名、型、enum）
- 開発フェーズの進捗が最新か
- 技術スタックに変更がないか

### 4. 修正ルール

- 実装を正とし、ドキュメントを合わせる
- 完了タスクは `[x]` / `~~取り消し線~~` にする
- フォーマットは既存スタイルを踏襲する
- 変更なしなら「差分なし、最新です」と報告して終了

### 5. 変更サマリーを報告

修正箇所を箇条書きでユーザーに報告する。
